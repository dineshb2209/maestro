  4. case class LoadConfig[A](errors: String, splitter: Splitter = Splitter.delimited("|"), timeSource: TimeSource = TimeSource.fromDirStructure, generateKey: Boolean = false, filter: RowFilter = RowFilter.keep, clean: Clean = Clean.default, none: String = "", validator: Validator[A] = Validator.all[A](), errorThreshold: Double = 0.05, charsetEncoding: String = TextLine.DEFAULT_CHARSET, splitSize: Long = 9L * 128 * 1024 * 1024) extends Product with Serializable

    Configuration options for load

    Configuration options for load

    errors

    The hdfs path used to store rows which failed processing.

    splitter

    The Splitter used to split rows into fields. Defaults to splitting on the "|" character.

    timeSource

    The TimeSource used to find the effective date for each row. Defaults to pulling the year, month, and day from the directory structure of the input file.

    generateKey

    If true, load will append a unique key to each row. Defaults to false.

    filter

    The RowFilter which scans each row after splitting, potentially discarding bad rows. Defaults to now filtering.

    clean

    The Clean function to be applied to each field. Defaults to trimming fields and removing non-printable, non-ASCII characters.

    none

    The field value denoting a None value for an optional field. Optional string fields are None only if they match this value. Other optional fields are None if they match this or the empty string. Defaults to the empty sring.

    validator

    The Validator to check the thrift struct to ensure it is correct. Defaults to no validation.

    errorThreshold

    The fraction of rows that can fail during the load step before the overall job should fail. Defaults to 0.05 or 5%.

    charsetEncoding

    The Scalding text encoding (Cascading charset) for parsing text files. Defaults to TextLine.DEFAULT_CHARSET, which is UTF-8. However, the default vaule of clean will remove non-ASCII characters, so consider an alternative like Clean.unicodeDefault if you need to preserve non-ASCII characters from the source.

    splitSize

    The size of each split in the job that processes the pipe returned by load. Defaults to 9 blocks, with the hope that this will result in 2-block parquet files.

  16. case class UploadConfig(localIngestPath: String, hdfsLandingPath: String, localArchivePath: String, hdfsArchivePath: String, tablename: String, filePattern: String = "{table}?{yyyyMMdd}*", controlPattern: Regex = ControlPattern.default) extends Product with Serializable

    Configuration options for upload.

    Configuration options for upload.

    Files are copied from the local ingestion path to the three destination paths (HDFS landing path, local archive path, and HDFS archive path). When files are copied to a destination path the effective date of the upload are added to the path. For instance: the actual location of files on HDFS might be $hdfsLandingPath/<year>/<month>/<day>/<originalFileName>.

    localIngestPath

    The local ingestion path where source files are found.

    hdfsLandingPath

    The HDFS landing path where files are copied to.

    localArchivePath

    The local archive path where files are archived.

    hdfsArchivePath

    The HDFS archive path, where files are also archived.

    tablename

    The table name or file name in database or project.

    filePattern

    The file pattern, used to identify files to be uploaded. The default value is {table}?{yyyyMMdd}*.

    controlPattern

    The regex which identifies control files. The default value is au.com.cba.omnia.maestro.core.upload.ControlPattern.default.

  17. trait UploadExecution extends AnyRef

    Push source files to HDFS using upload and archive them.

    Push source files to HDFS using upload and archive them.

    See the example at au.com.cba.omnia.maestro.example.CustomerExecution.

    In order to run map-reduce jobs, we first need to get our data onto HDFS. upload copies data files from the local machine onto HDFS and archives the files. The user can also use findSources and uploadSources, which allows them to inspect the data files before uploading them.

    The user can run parseHeader, parseTrailer, and parseHT on data files to retrieve and validate their metadata.
